# 写作模式参考

## 定义 → 思考 的融合模式

### 模式1：先定义，再消化

```
### 1.1 概念定义

> **定义：** [标准数学表达]

### 1.2 直观理解

[用自己的话解释]
[可以包含：
- 简化版本
- 比喻
- 图解描述]

### 1.3 小结
[一句话概括核心]
```

### 模式2：问题驱动

```
[从一个有趣的问题/悖论开始]

**问题：** [抛出问题]

**分析：** [逐步分析]

**结论：** [给出定义/定理]

**思考：** [个人理解]
```

### 模式3：演进式

```
[从一个简单的特例开始]
→ [逐步推广到一般情况]
→ [给出正式定义]
→ [讨论例外情况]
→ [个人感悟]
```

## 图解/直觉 的写法

### 1. 几何解释
```
[描述图形]
[例如： imagine two points in a plane...]

[解释几何意义]
```

### 2. 类比/比喻
```
[核心概念] 就像 [日常生活中的例子]
- 相同点：...
- 不同点：...
```

### 3. 一步步推导
```
让我们从[最简单的情形]开始：
1. 首先...
2. 然后...
3. 最后...
这样我们就看到了...
```

## 个人思考的写法

### 1. "我发现..."
```
笔者认为，这里最关键的一点是...
这意味着...
```

### 2. "直觉上..."
```
直观上，这个结论告诉我们...
从几何上看...
```

### 3. "延伸思考..."
```
如果...会怎样？
这个问题还可以从...角度考虑...
```

### 4. 适度吐槽
```
[可以加入适度的个人风格，如：
"这个证明，笔者并不想在考试中碰到"
"笔者粗浅的理解是..."
]
```

## 引用格式

### 书籍
```
**作者** (年份). *书名*. 出版社.
```

### 论文
```
**作者**. (年份). 论文标题. 期刊名, 卷(期), 页码.
```

### 课程/讲义
```
**课程名称**. 学校/机构.
```

### 网页
```
**标题**. URL. (访问日期).
```
